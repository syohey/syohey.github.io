Git Basics

2020/05/15

Tags: git

  1. $ git init to initialize a git repo.

  2. $ git config --list to show your git config.

  3. $ git add <file-name> to stage your file(s).

  4. $ git commit -m "<your-comment>" to make a commit.

  5. $ git push to save your work on your GitHub or GitLab or…?

  1. $ git status to show the current working repo’s status.

  2. $ git branch to show what branches you have.

  3. $ git branch <branch-name> to create a new branch.

  4. $ git checkout <branch-name> to work on your branch.

  1. $ git reset <commit> { --soft / --mixed / --hard } (Choose amongst these three. If --mixed, you don’t need to.) to reset your commit. Default is --mixed, so choose --soft or --hard. --soft resets the commit, but everything is still staged. --mixed resets the commit and also unstages everything. --hard resets the commit and discard all the change you made in your code.

  2. $ git checkout <commit> — You can also checkout the previous commits.

  3. $ git checkout -- <file-name> — This is especially useful when you accidentally deleted file becasue this command will restore it.

  4. $ git ls-files --deleted to show accidentally/intentionally deleted files.


In case you have already also accidentally commited with deleted files, these commands will save you??

  1. $ git rev-list -n 1 HEAD -- <file>

  2. $ git checkout <commit>^ -- <file>

  3. $ git log --diff-filter=D --summary
